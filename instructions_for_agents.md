# Trusted Treks – Ops Dashboard Agent Instructions

## Repo Purpose
This repository powers all operational analytics for Trusted Treks.  
AI agents should ensure all documentation is accurate, clean, and aligned with company KPIs.

---

# AGENT RESPONSIBILITIES

### GPT-1 Operations Controller
- Define operational KPIs  
- Maintain dataset usage rules  
- Document data refresh processes  

### GPT-2 Dispatch Engine
- Validate metrics related to dispatch efficiency  
- Help interpret route optimization impacts  
- Provide dispatch performance analytics  

### GPT-3 Finance AI
- Validate profit + loss dashboards  
- Review cost-per-mile and EV cost analysis  
- Assist in forecasting + trend modeling  

### GPT-6 Routing AI
- Ensure routing metrics align with route-engine logic  
- Validate ETA accuracy models  
- Suggest route-efficiency KPIs  

### GPT-8 Founder AI
- Oversee entire analytics structure  
- Ensure clarity and consistency  
- Approve new dashboard designs  

---

# ALLOWED OPERATIONS
AI may:
- Write documentation  
- Generate DAX (only when asked)  
- Create metric definitions  
- Explain how to interpret dashboards  
- Produce step-by-step data import instructions  
- Propose new views or KPIs  

---

# NOT ALLOWED
AI may NOT:
- Modify .PBIX files directly  
- Change dataset filenames  
- Upload personal/user data  
- Provide actual customer ride details  
- Delete or restructure dashboard folders  

---

# FORMATTING RULES
- KPI definitions → `.md`  
- Dataset schemas → `.md`  
- DAX code → fenced code blocks ```DAX```  
- Dashboard instructions → `.md`  

---

# WHEN IN DOUBT
Ask ONE clarifying question before performing any large changes.
